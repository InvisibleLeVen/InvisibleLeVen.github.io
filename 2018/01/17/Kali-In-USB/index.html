在install system那一步的时候有个Failed to copy file from CD-ROM. Retry?蜜汁错误。解决不了

u盘装kali的教程。发现找到了一些新方法，在VM上装，(当时在学校没有仔细看，直接在自己电脑上装的)。教程
链接是http://blog.51cto.com/zdzhu/1574543。网上很多类似的，然而一开始看的是用Universal-USB-installer
直接在u盘上做镜像。然后bios启动直接安装。
嗯这个直接在VM上装的方法保险多了。选择硬盘的时候注意一下就好了。
于是便按照这个步骤一路下去。。。。。
等利用VM把系统成功的装在U盘里时，重启PC。设置bios。我的是dell的笔记本，直接进入bios setting设置
security boot 改为disable。然后再把Boot List Option改为Legacy
F10.开机按F2设置U盘启动。然后便进入到了grub引导界面
第一个是kali GNU/linux，第二个是Advanced options for kali GNU/Linux.
直接选择第一个。遇到了Target filesystem doesn’t have requested /sbin/init错误

大概导致这样的原因就是，linux中硬盘名称分配是按照/dev/sda、/dev/sdb之类的序号分配。有三种情况可能导致
分配的序号发生变化。无法启动系统。通过uuid来标记磁盘。则不会发生这种错误，下面要做的也就是这个
按照博客所描述的。在initramfs模式下输入blkid查看所列举出的磁盘，找到U盘系统TYPE=’ext4’的区

在评论区看到了令人高兴的回答。有相应的解决方案
I think I solved the problem by adding nouveau.modeset=0 to the kernel parameters. The error messages at boot are gone, and I can execute lspci without my laptop crashing.
内核参数。查了一下nouveau是一个linux下针对于NVIDA的第三方显卡驱动。所以这里的错误应该是没有合适的显卡驱动造成的
那么先加那个内核参数忽略nouveau驱动。便可进入kali。然后再
sudo add-apt-repository ppa:graphics-drivers/ppa
sudo apt-get update
sudo apt-get install nvidia-361
加不加那个内核参数便无所谓了。
